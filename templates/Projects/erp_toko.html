My family runs a retail / distribution business in Nusa Tenggara Timur, Indonesia. The traditional business model, which includes manual recordkeeping and hand-written receipts was getting messy, and we were having trouble keeping up with the scale of business.

As a result, a lot of potential business was lost, not to mention the time investment required to manage all the functional areas manually.

Another problem is that the workforce in rural Indonesia are largely unfamiliar with computers, so using ready-made software solutions present a heavy learning curve.

The interface had to be as simple and intuitive as possible–only the bare necessities.

Iteration 1: Smart Excel Book with VBA

Improvised UI Using VBA Forms

Mock Database with Excel Table

Starting in 2021 with an Excel (Visual Basic for Applications) based program, I had slowly computerized functional areas of my family business.

It was initially just a simple Point of Sale (POS) interface, the barebones implementation of technology had, to my surprise, drastically improved queueing times at the store.

Though functional, this approach had obvious flaws. Using excel meant that I had to make the built in methods function as a database. At this point I had only basic programming capabilties, so most of the business logic was flawed to say the least.

Online capabilities were also severely limited. I had some success implementing services like Dropbox and Google Drive to store the Excel file and update it when the user is idle, but it was unreliable to the point that I decided to seek another solution entirely.

Iteration 2: Django Web Application

POS Interface with Autocomplete Functionality

Barebones Inventory Management using Datatables

As I learned more in university, I reworked the system into a web-based application, using business principles to create a Stock-keeping system, which included Inventory and Purchasing.

This allowed me to track Stock-Keeping Units (SKUs) more efficiently and not have to rely on shoddy information when asked about a certain product’s availability–a common issue in traditional businesses.

Looking back at this now, the UI feels too bright and exaggerated, not to mention the project is poorly structured, so whenever I went to make changes or add new features, I had to search through the mess for a good while before getting my bearings straight.

However flawed it is in hindsight, it was still a major improvement over the old iteration. It can now be accessed through mobile and desktop, and I can remotely control the database and help clean up any input errors.

Iteration 3: Remastered Web App

Going into my 5th semester, I wanted to implement a Business Intelligence (BI) module and touch-up the User Interface to make it more responsive and easy to use. The BI module allows me to backtrack SKUs and find where mistakes in data input might have happened.

I also implemented stochastic calculations to help project demand and make more informed purchases. During this rework I also separated the project structure for better code readability, and more efficient data transfer.

This is the most recent and (for the time being) penultimate version of this project. Instead of an overview, I thought I would showcase the app's full list of features here while I'm at it.

Challenges 

Aggregate Query with Time Range

I realized when working on the VBA application that certain tasks such as aggregation and deletion are annoying to implement without the use of a database. This, and cross-platform accessibility was my main reason for transitioning into the web application.

During the early stages of building the web application, I had trouble with runtime. Queries get bigger and bigger as the database grows–I had to adjust the queries to make them more efficient.

Printer Controller Script

Setting up the printers for the POS system was also a challenge. I had to write custom Bash scripts and install miniature computers to act as mediators between the printers and my web application.

This included retrieving the data (SSH Tunneling), pre-processing, and then encoding it for the specific kind of printer–for larger transactions we used a dot-matrix printer, which had a different script compared to the thermal receipt printer.

Front-End (Js) Logic

Back-End (Python) Logic

To increase the efficiency of my web application, I thought it would be a good exercise to replicate the design principles of React and Flutter–only updating parts of the page that needed to be updated.

This required an extensive rework, and to make direct links work I had to code in a bunch of back-end logic to make pages able to render fully or partially according to the use case.

Things I Learned 

This is a long, ongoing project—so to say that this project encompasses the bulk of my skill set wouldn’t be an overstatement.

The highlights for me have been translating real-world business processes into the program, while identifying and creating solutions only possible through IT.

I also learned a great deal about database management and some hardware stuff while working on the POS system.

I reckon I also became better at management through implementing the web app in the field, teaching family and employees alike, while also taking feedback and revising along the way.
