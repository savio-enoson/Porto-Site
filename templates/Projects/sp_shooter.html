Space Shooter Game

With LibGDX and OOP Concepts

Problem 

This was the final project for my object oriented programming course. We were tasked with creating a game which implements OOP concepts using LibGDX–a Java library for creating games on multiple platforms. There were multiple types of games to choose from, and I ended up picking a Space Shooter type game.

 Source Code  Gameplay

Solution 

I made 3 main or superclasses for this game, Ship, Weapon and Floater. Ship encompasses the PlayerShip and EnemyShip, which is further divided into 2 subclasses, NormalEnemy and HardEnemy (a more sporadic movement pattern). Floaters are objects that travel down through the screen.

There are 2 child classes, Heal and Asteroid. Heal does as expected, and Asteroids make the player ship take damage when hit. Next I implemented 3 different weapon types to switch up the playstyles. Pistol, Shotgun, and Laser.

These use the Weapon Interface to connect with the Ship dynamically. Enemies spawn with pistols by default, but as the game goes on Polymorphism is used to cast the weapon to either Shotgun or Laser to increase difficulty.

The usage of Inheritance allows me to efficiently loop through entities in the game and implement the game logic in a more cohesive manner (logic for each type is within each subclass’s method). I also implemented a leaderboard system which uses serialization–and added a leaderboard page for players to compare their scores.

Challenges 

Learning the LibGDX library was made a bit more complicated by the lack of community resources, I guess the engine is not as popular as other ones like Unity.

I also had trouble finding places to implement OOP concepts like Interfaces and Polymorphism, although they eventually fell into place with the Weapons system.


Things I Learned 

This hands-on experience really helped me internalize the concepts in OOP. It was also very satisfying to come up with solutions that use everything I had learned in the course in a way that did not feel forced.
